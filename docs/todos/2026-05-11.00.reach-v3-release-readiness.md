## Detailed Plan
Priority and execution order:

1. Invalid DLsite response hardening (#30) - Critical, medium difficulty. Add stream/metadata/content validation so HTML/404/error pages cannot be persisted as successful archives. This is release-safety work and should be done before more library-state features.
2. Version/about info (#4) - High priority, easy. Add a small support-oriented version/about surface so logs and user reports can identify the build.
3. Manual downloaded-state marking (#33) - High priority, medium. Let users mark an owned work as already downloaded by selecting an existing local folder.
4. Flexible RJ-code detection (#32) - High priority, medium. Add deterministic parsing for RJ/VJ/BJ/etc. codes in local folder names and tests before scan/import.
5. Local-only / not-owned scan/import (#32) - High priority, hard. Add storage and UI for products present locally but not owned by synced accounts.
6. Account removal behavior - Medium priority, medium. Define and implement remove semantics for account source rows without accidentally deleting cached work data needed by other accounts.
7. Serial-number presentation policy - Medium priority, medium. Decide UI/copy behavior and sensitive-storage rules before exposing serial values.
8. Product detail view - Medium priority, medium-hard. Provide a place for full metadata, owners, paths, serials if enabled, and diagnostics.
9. Packaging/release validation - Critical, hard. Validate app build, app data dirs, credentials, logs, download/open/delete flows, and archive extraction on target platforms.
10. Tags/hidden/search expansion (#18/#19/#20/#22) - Lower for first v3 preview, medium-hard. Implement before full v3 milestone closure if required, otherwise explicitly defer past preview.
